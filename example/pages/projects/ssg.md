## Static Site Generator

This very site is generated by a static site generator built entirely from Concept Design primitives.

### Architecture

The SSG is composed of **9 concepts** and **11 sync rules** totaling under 2,000 lines of code:

```
Commanding     — CLI command lifecycle
Building       — Build progress tracking
Filing         — Filesystem I/O
Frontmattering — YAML metadata parsing
Formatting     — Markdown-to-HTML conversion
Routing        — File path → URL route derivation
Layouting      — Template composition (components, slots, loops)
Collecting     — Collection membership for list pages
Publishing     — Output snapshots and stale-file cleanup
```

### Build Pipeline

```
issue("build")
  → start build
  → clear state
  → configure routing
  → scan layouts → read → define templates
  → scan content → read → parse frontmatter
      → render markdown
      → derive route
      → collect metadata
      → apply layout
      → write to disk
  → regenerate index pages
  → clean stale output
  → succeed
```

### Key Design Decisions

**Layouts optional.** If no layout template is defined, raw HTML is written directly. This means you can start with plain markdown and add layouts later.

**Route collision detection.** If two source files derive the same URL (e.g., `about.md` and `about/index.md`), the build fails with a clear error rather than silently overwriting.

**Stale output cleanup.** Files removed from the source are automatically removed from the output on the next build. No stale pages left behind.

**Real YAML.** Frontmatter uses the `yaml` package for full YAML compliance — arrays, nested objects, CRLF line endings, and proper quoting.

### Running It

```bash
bun run src/main.ts build \
  --source example/pages \
  --output example/dist \
  --layouts example/layouts
```

### Tests

98 tests cover every concept action, sync cascade, and the full build pipeline. Tests use isolated in-memory state — no filesystem dependency except for the Filing concept's integration tests.
