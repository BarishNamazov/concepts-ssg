## Building a Static Site Generator With Concepts

This site is generated by a static site generator built entirely from concept-design primitives. Here's how it was constructed.

### Step 1: Identify the Concepts

A static site generator needs to:

1. Accept a build command from the CLI
2. Discover source files on disk
3. Parse frontmatter metadata
4. Convert markdown to HTML
5. Derive URL routes from file paths
6. Apply layout templates
7. Organize content into collections
8. Write output files to disk
9. Clean up stale files from previous builds

Each of these becomes a concept:

| Need | Concept |
|---|---|
| CLI boundary | `Commanding` |
| File discovery & I/O | `Filing` |
| Metadata extraction | `Frontmattering` |
| Markdown rendering | `Formatting` |
| URL routing | `Routing` |
| Template composition | `Layouting` |
| Collection tracking | `Collecting` |
| Build lifecycle | `Building` |
| Output snapshots | `Publishing` |

### Step 2: Specify Each Concept

Each concept gets a specification: name, type parameters, purpose, principle, state, actions, and queries. For example, `Formatting [Target]`:

```
purpose
  convert structured markup formats to HTML for web presentation

principle
  after rendering markdown source through the formatter,
  the resulting HTML is preserved and queryable independently

state
  a set of Entries with a source String, format String, and html String

actions
  render (target, source, format): (target, html)

queries
  _getHtml (target): (html)
```

### Step 3: Write the Syncs

Eleven sync rules compose the pipeline. Here's the core flow:

```
Commanding.issue("build")
  → Building.start
  → Filing.clear + configure routing
  → Filing.scan(layouts) → read → Layouting.define
  → Filing.scan(content) → read → Frontmattering.parse

Parse results fan out to:
  → Formatting.render (markdown → HTML)
  → Routing.derive (path → URL)
  → Collecting.collect (metadata → collections)

Render + route both ready:
  → Layouting.apply (template + content = page)
  → Filing.write (page to disk)

All pages written:
  → Building.complete
  → Filing.cleanOutput (remove stale files)
  → Commanding.succeed
```

### Step 4: Test in Isolation

Each concept is tested independently. Syncs are tested with fresh concept instances created per test. The full pipeline is verified by building a real site and checking output files.

### The Result

A fully functional static site generator in under 2,000 lines of concept and sync code. Every piece is independently understandable, testable, and reusable.
